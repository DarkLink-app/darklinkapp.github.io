How it works

Zero server-side trust.

DarkLink is designed so that a compromised server reveals nothing. Encryption is handled entirely client-side — the server only sees opaque envelopes, never the content.

End-to-end encryption

Every message is encrypted with P256 ECDH + AES‑256‑GCM, per device. The server stores only ciphertext and deletes it after delivery.

Tor-only transport

Every DarkLink server is a .onion address. No open ports on the Internet, no visible IP, no DNS. Access is exclusively through the Tor network.

Renewable identities

Every user has a public identity that can be rotated independently of their account. Rotation updates existing contacts without interrupting conversations.

Inter-server federation

Two DarkLink servers can be federated: users add each other via QR code and message directly, server to server, over Tor.

Simple self-hosting

A single script installs Docker, configures Tor, generates the .onion addresses and starts the server. No ports to open. Runs on any Linux machine.

Automatic .onion rotation

Admins can rotate .onion addresses manually or automatically on a schedule. Tor restarts on its own and federated contacts are notified automatically.

Open source

Host your own

DarkLink is fully open source. The source code, install script and documentation are available on GitHub. A working server up in minutes on any Linux with Docker.
